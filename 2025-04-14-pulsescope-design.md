# PulseScope 产品设计方案

**日期：** 2025-04-14  
**版本：** MVP v1.0  
**状态：** 待实施

---

## 1. 产品定位

**一句话定位：** 面向大宗商品与化工企业的"事件驱动型供应链风险推理引擎"。用户订阅目标企业后，系统持续监测全球地缘、航运、期货、政策动态，主动输出"某事件 → 对某企业某业务线 → 产生某类风险"的结构化分析。

**核心价值：** 把"地缘事件如何影响具体企业"这个黑箱打开，提供可验证、可行动的动态风险情报。

---

## 2. MVP 边界

### 2.1 范围以内
- **首发行业：** 化工/大宗商品（以聚氨酯、石化、油运为起点）
- **数据源：**
  - 地缘新闻（Tavily / NewsAPI / 自建爬虫）
  - 航运 AIS（船讯网 / MarineTraffic 等公开/半公开数据）
  - 期货/大宗商品价格（Wind / 公开 API）
  - 企业公告（巨潮资讯网 / 交易所公告）
- **输出形态：** MCP Skill / Native Skill / REST API，返回结构化 JSON/Markdown 风险报告
- **推理深度：**
  - L1 事件识别：什么发生了
  - L2 产业链映射：影响谁的哪个环节
  - L3 风险定级：高/中/低，附理由与建议关注指标

### 2.2 范围以外（后续迭代）
- 实时毫秒级告警
- 股价/利润量化预测
- 跨行业通用浅层覆盖
- 可视化大屏 SaaS 前端
- 社交媒体情感分析
- 卫星遥感数据

## 3. 目标用户

| 用户群体 | 使用场景 | 付费能力 |
|---------|---------|---------|
| 期货研究员/交易员 | 提前发现事件驱动的投资机会或对冲需求 | 高（已采购 Wind/Bloomberg） |
| 大宗商品贸易公司风控部 | 监控供应商/客户的航运、地缘、价格波动风险 | 高（数据预算充足） |
| 化工企业采购经理 | 预判原材料供应中断或运费飙升 | 中高 |

**MVP 阶段聚焦：** 期货私募和大宗商品贸易公司风控部门。

### 4.1 核心技术栈
- **数据管道：** Python + Airflow / Temporal
- **向量数据库：** Qdrant / Pinecone（新闻/公告语义检索）
- **图数据库：** MVP 阶段使用 NetworkX / 轻量 JSON 图谱，验证后迁移 Neo4j
- **LLM：** Claude / GPT-4（推理链）+ BGE-m3（本地 embedding）
- **Skill 封装：** FastMCP（MCP Server）或 Native Skill

### 4.2 MVP 技术路径
1. **Week 1-2：** 搭建新闻/公告爬虫，验证事件抽取质量
2. **Week 3-4：** 构建化工企业种子知识图谱（10 家核心企业）
3. **Week 5-6：** 实现 LLM 推理链，输出结构化风险报告
4. **Week 7-8：** 封装成 MCP Skill，找 3-5 个种子用户测试
5. **Week 9-12：** 根据反馈迭代，接入航运/期货数据

## 5. 商业模式

### 5.1 收入模型

| 层级 | 客户 | 定价 | 预估 ARPU |
|------|------|------|-----------|
| **Tier 1: API/Skill** | 期货私募、大宗商品贸易商、化工企业风控 | 按企业订阅数：¥500-2000/企业/月 | ¥1000/月 |
| **Tier 2: 定制报告** | 投资机构、咨询公司 | 单份深度报告 ¥5000-20000 | 项目制 |
| **Tier 3: SaaS（远期）** | 个人投资者、中小企业 | ¥99-299/月 | ¥150/月 |

**MVP 阶段只做 Tier 1。**

### 5.2 获客策略
1. **内容获客：** 在雪球、知乎、公众号发布"地缘事件对 MDI 产业影响分析"类文章
2. **种子用户：** 直接联系大宗商品贸易公司风控/研究员（LinkedIn、行业社群）
3. **API 集市：** 上线阿里云市场、火山引擎等 B 端 API 平台

### 5.3 成本结构（MVP 3 个月）
| 项目 | 月成本 | 说明 |
|------|--------|------|
| LLM API | ¥2000-5000 | 推理链调用 |
| 数据采购 | ¥1000-3000 | 新闻 API、航运数据 |
| 服务器/向量库 | ¥1000 | 轻量部署 |
| 人力 | 2-3 人 | 全栈工程师 + 内容/BD |

## 7. 里程碑

### 3 个月 MVP 里程碑
- [ ] M1：能稳定输出 10 家化工企业的每日风险扫描
- [ ] M2：有 3 个付费种子用户（哪怕是按次付费）
- [ ] M3：至少有一次"提前预警被市场验证"的案例

### 最小团队配置
- **全栈工程师 × 1：** 数据管道、API、Skill 封装
- **内容/BD × 1：** 行业分析文章、种子用户拓展
- **产品/策略 × 1（你）：** 产品定义、推理链 Prompt 工程、战略决策

## 8. 核心风险与应对

| 风险 | 应对策略 |
|------|---------|
| 数据获取不稳定 | 多源备份，核心数据签年度合同锁价 |
| LLM 幻觉导致推理错误 | 强制结构化输出 + 每一步引用来源 + 种子报告人工复核 |
| B 端销售周期长 | 同时做 Tier 2 定制报告，用项目制现金流养活产品 |
| 竞争抄袭 | 壁垒在知识图谱精度与行业 know-how 积累速度 |
